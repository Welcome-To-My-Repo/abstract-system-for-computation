Data

The environment assumes at least two capabilities: the ability to hold data temporarily while a transformation is being applied and the ability to store data in between transforms. Two formats are required, one for performing transformations on data and one for storing data. The Word is the format for transforming data. The Byte is the format for storing data.

If multiple storage or transform formats are available to the environment, then they may be provided in ASC as data formats in addition to Words and Bytes.

The Byte

The Byte corresponds to a single unit of storage. All entities in the Environment may be stored using one or more bytes. A Byte format is assumed to be capable of representing some number of whole, positive values. A Byte must be convertable to a Word value.

In the case of multiple viable formats being available to the Environment, the Byte must correspond to the smallest uniquely identifiable storage unit available to the Environment. The ability to uniquely identify a single unit may include indirection.

Size

The number of Bytes required to completely store an entity is referred to as the "size" of an entity. The size of an entity is considered a constant value and does not require computation. A program may query the size of any entity.

Location

The Environment assumes that the available storage resources collectively provide a countable number of Bytes. Each byte may then be assigned a unique whole, positive number. The Environment assumes this numbering is arbitrary and does not correspond to the physical position of bytes in a storage medium. The unique number identifying a Byte is considered a constant value and does not require computation. A program may query the identifying number of a Byte.

The Word

The Word corresponds to the the implementation specific format used for transforming data. The Word format must allow the basic arithmetic operations, addition, subtraction, multiplication, division yielding a quotient, and division yielding a remainder. These operations shall follow the natural behavior of arithmetic over the set of Integers as far as the Word format is capable of representing Integer values. If any of these operations are available to the environment as part of the implementation specific Word format, then the environment is not required to simulate those operations and may use the ones supplied with the format. This is conditional on the provided operations being at least as accurate as simulated operations.

In the case of multiple viable formats being available to the Environment, the Word must correspond to the format that offers the most complete set of arithmetic operations. Additionally, the format which is most natural to the computer must be used. That is to say a format which is preferable for the Environment may not be the format most conducive to efficient computation. This may disqualify some formats which would otherwise be judged suitable. The ultimate choice of format is completely implementation dependent and is purposefully left vague.

Compound Data

Words, Bytes, and all other Types may be used to form compound formats. A compound data format is treated as a single unit of data, but transformations are still performed on individual Word and Byte elements. Each individual component in a compound data format is referred to as a "member".

Constructing Compound Data

Compound data is constructed by declaring a list of one or more individual formats. Each individual format is given an index beginning with zero for the first member and increasing by one for each additional member. Individual members are accessed by performing a subscript operation.

Size

The size of a unit of compound data is the sum of the size of each member. The size may be a larger number of bytes to account for any additional spacing required by the Environment

Location

A single unit of compound data is assumed to occupy a contiguous series of Bytes in storage. Depending on the size of a compound data unit, a contiguous storage segment may not be available. If a contiguous storage segment is not available, then constructing a compound data unit of that size must be rejected.
