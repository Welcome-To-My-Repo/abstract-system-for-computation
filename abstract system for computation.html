Abstract

This document describes the notation and semantics of an Abstract System for Computation, abbreviated as the name "ASC". It provides advanced features which are useful for expressing and reasoning about complex algorithms and computer programs.

Introduction

Properties of an Abstract System for Computation

An Abstract System for Computation is a concept which is both specific to this document and defines the philosophy involved with the creation of ASC. The key consideration is that an abstract system for computation is a higher abstract concept which the programming language, ASC, is only intended to represent. There may very well be other, alternative systems for programming or mathematical reasoning which can also be described as an abstract system for computation, but they are not defined here. In fact, the title of this document uses "An" instead of "The" to make this point explicit. The programming language described in the following chapters is being proposed as one of many possible manifestations of an abstract system for computation.

An abstract system for computation is mainly a system for mathematical and logical reasoning. The manner in which it represents structured information and algorithms is based in mathematics rather than pieced together nominally. Every component is formalized or founded in an external theory.

An abstract system for computation is different from mathematics because it addresses different concerns. It is a lower level of abstraction than most mathematics. Classical mathematics is concerned with the correspondence between problems and answers. By contrast, abstract systems for computation are concerned with how an answer can be obtained and if solving a problem is possible.

Syntax

ASC programs are represented by a sequence of characters which is converted into a corresponding sequence of symbols using a set of grouping rules. Symbols are then mapped to program semantics using a set of notation rules. Notations may be defined as part of the program itself and can affect how symbols and expressions are identified.

Symbols

Symbols consist of sequences of visible characters delimited by blank, unprintable, and control characters. For the ASCII table, character codes 0-31 and 127 are delimiting characters. Every other character may be included in a symbol.

Examples of Symbols

The character sequence
abc def ghi
corresponds to the three symbols abc, def, and ghi.

The character sequence
a&* &9)(li afj i3#afs
corresponds to the four symbols a&*, &9)(li, afj, and i3#afs.

Note that no distinction is made between symbolic and script characters. This allows for names which may include symbols.

Regarding Symbol Alternate Encodings

If alternate character encodings are used, the rules for grouping characters into symbols should be such that any visible characters compose symbols and any non-visible characters delimit symbols. It may be necessary to transparently ignore certain characters such as formatting codes which are technically invisible, but change the textual appearance of a visible character. Alternatively, additional rules may be necessary to decide if a character is delimiting or used in symbols. The task of interpreting and applying these rules to alternative character encodings is left to the implementation.

Formalization of Types

ASC utilizes a system of types which is founded in the description of geometric objects and their relations. In this way, types can be extended to reflect any theory of geometry. Judgements about type can then be enhanced by building on pre-existing relationships established in other fields of mathematics.

Additionally, this system of types is neither complete nor consistent. In fact, it explicitly allows for contradictory or absent information. The intent is that consistency and completeness are sacrificed for greater expressiveness. Mechanisms for handling illogical information can be used to reason about programs or environments which present illogical states and behavior. This kind of system acknowledges that a judgement in real life may depend on multiple gradual stages of reasoning or even missing information.

Definition of Type

  1. A type is some abstract geometric form with features.
  2. A feature is some consideration about a geometric form such as dimension, size, shape, or any arbitrary term.
  3. Every type has a countable number of features from none to infinite.
  4. Multiple types may have the same feature. A feature shared between two types is called a common feature.
  5. There is a single type which has no features and it is the Null type.
  6. There is a single type which has every possible feature and it is the Any type.

Type Relationships

  1. Consider two types where every feature between them is common. These types are equivalent.
  2. Consider two types where every feature between them is not common. These types are disjoint.
  3. Consider two types where at least one feature is common. These types are alike.

Type Structures

  1. A unit is a type which has a single feature and consequently serves to identify that feature.
  2. A function is a type which represents a transformation between two collections of types.
  3. A composition is a type which has a common feature for every feature present in a collection of types.
  4. A superposition is a type which is possibly equivalent to one of a collection of types.

These structures represent various correspondences which appear as types, but express properties that are external to the definition of types as it is given here.
