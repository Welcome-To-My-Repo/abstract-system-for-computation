Overview

The Abstract System for Computation (abbreviated as ASC) is a tool designed for describing, performing, and reasoning about computation. It exists as a written notation and an environment for verifying and interpreting the notation. This document outlines the capabilities and requirements of a verified implementation of ASC.

ASC is a minimal language designed to be as general as possible so it can be adapted to the widest possible range of computational scenarios. It provides a reduced set of constructs: functions, collections, types, and evaluation context. Every entity is represented as some computable value in ASC, including programs themselves.

Syntax

ASC syntax represents the correspondence between a written program and its computable model. Syntax is composed of a sequence of characters divided into sub-sequences called Symbols. These symbols are grouped into Expressions. Expressions are then nested in a structure representing the computable model of a program.

Character Encoding

Each character in ASC must correspond to a unique numerical value in an encoding scheme. The specific encoding scheme used depends on the implemntation. If no encoding scheme is used, an ASC program may assume the default encoding is the Unicode standard encoding.
